# Trigger Evaluation

Eyeball this list whenever the `description` field in SKILL.md changes. A correct skill fires on every "should trigger" query and on none of the "should not."

## Should trigger

- "Can you stress-test this plan?"
- "Adversarial review of the plan above."
- "Poke holes in this plan before I build it."
- "Red-team this plan."
- "What's wrong with this plan?"
- "Am I missing anything in this plan?"
- "Review this implementation plan critically before I start."
- "Find flaws in the approach you just proposed."
- "Harden this plan."
- "Challenge the plan — what would break?"

## Should NOT trigger

- "Review this PR."
- "Review the code I just wrote."
- "Review my commit message."
- "Debug this error."
- "What does this function do?"
- "Review our Q3 product strategy."
- "Critique this blog post draft."
- "Look over this config file."
- "Run the tests."
- "Summarize the plan above."
